# メニュー管理ツール

## デモ
https://kaniten0410-lang.github.io/recipe-summary/

## プロジェクト概要
あらゆるレシピURLをブックマークしていたが量が多く管理が難しくなってきたため、
一括で管理や編集・検索・ランダムでメニュー決めができるツールを作成した。

- 対象ユーザー：自分

## 機能要件
- 登録しているレシピを一覧で表示
- 条件を絞り込みランダムで一件出力
- データの全件表示・追加・更新・削除

## 非機能要件
| 項目 | 内容 |
|---|---|
| パフォーマンス | 一覧表示は体感で遅延なく表示 |
| セキュリティ | Googleログイン認証によりデータ管理ページを保護 |
| 対応環境 | PC・スマートフォン（PWA対応） |

## 画面一覧
- メインページ
- レシピ一覧
- メニュー決め
- データ管理
- 全データ閲覧
- データ追加
- データ編集

## 技術構成
| 技術 | 用途 |
|---|---|
| HTML / CSS / JavaScript | フロントエンド |
| Firebase (Firestore) | データベース |
| Google Apps Script | OGP・タイトル取得プロキシ |
| GitHub Pages | ホスティング |


## 制約・除外事項
- レシピ情報の自動更新は対象外
- ユーザー認証・ログイン機能は実装しない
- 複数ユーザーでの共有・共同編集は対象外

## 工夫した点
- OGP画像を取得し視覚情報を追加
- URL無しの定番料理も登録できるよう対応
- 更新・削除時に現在の登録情報を表示
- PWA化によりスマートフォンからも使いやすく対応
- CORSを回避するためGASをプロキシとして活用し、OGP・タイトル情報を取得した

## 苦労した点
- フレームワーク不使用でHTML・CSS・JavaScriptのみで構成した
- AIツールを活用しながら独学で実装した
